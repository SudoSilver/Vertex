# Security Policy

## Supported Versions

Security updates are provided only for the latest stable version of Vertex.

| Version | Supported |
|---------|----------|
| latest  | ✅       |
| older   | ❌       |

---

## Reporting a Vulnerability

If you discover a security vulnerability in the Vertex language, compiler, runtime, or standard library, please report it responsibly.

### Contact
Please report issues via:
- GitHub Issues (preferred): https://github.com/DomioKing653/Vertex/issues

For sensitive security issues (if available):
- Use GitHub Private Vulnerability Reporting

---

## What to Include

To help us investigate and fix the issue quickly, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Expected vs actual behavior
- Vertex version affected
- Operating system and environment details
- Minimal reproducible example (if possible)

---

## Response Time

- Initial response: within 3–7 days
- Fix timeline depends on severity:
  - Critical: as soon as possible
  - High: next releases
  - Low: handled in backlog

---

## Scope

This security policy applies to:

- Vertex compiler
- Vertex runtime / interpreter
- Build tools and package ecosystem (if applicable)

---

## Out of Scope

The following are generally not considered security vulnerabilities:

- Feature requests
- Crashes without security impact
- Performance issues
- Incorrect usage of the language without exploitability

---

## Security Best Practices (Users)

We recommend:

- Always using the latest version of Vertex
- Avoiding execution of untrusted code in production environments
- Validating all external input data
